Formats de données usuels dans un tableur

  • Nombre entier (ex. 23 ; pas de virgule).
  • Nombre décimal (ex. 26,3 en français — utiliser la virgule).
  • Pourcentage (ex. 45 %).
  • Date et heure (ex. 2025-08-08 14:30).
  • Texte (ex. « valeur suspecte », « Salle techno »).
  • Booléen (VRAI/FAUX) — utile pour les filtres.

Excel : Accueil → Nombre. Calc : Format → Cellules… → Nombre.

Activité 2 – Tableur (découverte) : saisir, trier et calculer une moyenne

Données à saisir (températures en °C) : 25, 23, 29, 28, 27, 30, 22.

Étapes – Excel (FR)

  1. Saisir une colonne Température (°C) avec les 7 valeurs (une par ligne).
  2. Trier la plage : Données → Trier A→Z (ordre croissant).
  3. Calculer la moyenne : =MOYENNE(A2:A8).
  4. Mettre en forme : Accueil → Mettre sous forme de tableau (ou bordures + en-tête en gras) ; format Nombre (1–2 décimales).

Étapes – LibreOffice Calc (FR)

  1. Saisir la colonne Température (°C).
  2. Trier : Données → Trier… → Ordre croissant.
  3. Calculer la moyenne : =MOYENNE(A2:A8).
  4. Mettre en forme : Format → Cellules… → Nombre (1–2 décimales) + bordures si besoin.

Astuce : pour afficher 2 décimales, formater la cellule en « Nombre » (2 décimales).

Valeurs triées : 22, 23, 25, 27, 28, 29, 30

Moyenne : (25+23+29+28+27+30+22)/7 = 184/7 ≈ 26,29 °C

Imagine que tu as collecté et trié les données d’humidité de la serre pendant une semaine. Explique comment ces informations permettront de programmer le microcontrôleur Arduino afin d’automatiser l’arrosage. Que faut‑il prévoir pour éviter un arrosage excessif ?

Correction possible : une fois les données analysées, on identifie un seuil d’humidité en dessous duquel les plantes doivent être arrosées. On programme alors l’Arduino pour comparer la valeur lue par le capteur à ce seuil : si l’humidité est inférieure, la pompe s’active, sinon elle reste arrêtée. Pour éviter un arrosage excessif, on peut ajouter un délai minimum entre deux arrosages et tenir compte de la météo (capteur de pluie) ou de la réserve d’eau. Il est également possible d’envoyer les données à un tableau de bord pour surveiller le fonctionnement.

Méthode : interpréter les données pour définir des seuils et des consignes ; programmer le système de commande en conséquence; prévoir des dispositifs de sécurité (temporisation, capteur de niveau) pour optimiser l’arrosage.
